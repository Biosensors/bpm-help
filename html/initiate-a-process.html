Overview

The initiate process function allow you to search all the available process template , and use it to initiate new process.

Initiate Process

  1. Click on "Initiate A Process" menu link
  2. Click on "My Draft Processess" if you have any draft process yet to submit. Otherwise, proceed to step 3 to search for template
  3. Click on "Search Template" tab and enter/select your search criteria
  4. Click on "Search" button
  5. Search result show list of available template based on search criteria
  6. To initiate process, click on the "New Process" link on the record
    • Contiunue by referring to Next Section - Enter Process Details
  7. For frequently used template, you may save it by clicking on the "Save To Favorite" button.
  8. To access the saved template, kindly click on the "My Favorite Templates" Tab.

Enter Process Details

  1. In the process detail page, enter "Subject"
    • This will be the email subject for email notification triggered to approver / action taker / acknowledger
  2. Enter on-behalf of (optional)
    • Indicate if process is raised on behalf of others . Else , leave it as blank
    • Depend on template setting, process might get routed to on-behalf user first for confirmation before proceed further for approval/action
    • For process that requires initiator's functional reporting for action (e.g. reporting manager, functional head and etc), it will be routed to on-behalf user reporting instead of reporting of user who submitted this process
  3. Enter process "Start On"
    • This serve as a schedule feature where process will kick start only on the user specified date
    • Depend on template setting, field might be disabled for particular template.
  4. Enter initiator input form
    • Input form will be rendered accordingly based on the template setting
    • "Submit" button will be enabled only once all the mandatory fields are provided
    • For attachement upload, please first save the process as draft and the upload function will be shown
  5. Review Person Responsible For The Process (e.g. Approver / Action Taker / Acknowledger) - see next section

Review Person Responsible For The Process

  1. Review by click on each of generated tab
    • This section will be auto generated based on the configured template and also info provided in form by initiator
    • Do feedback to template owner if you found that the process or populated person rensponsible is incorrect
  2. Review who is the person in charge of each of the process
    • There might be case where you need to input the rensponsible person yourself :
      • when template is configured in such way where initiator need to select who is the responsible approver/action taker/acknowledger
      • when there is multiple users found for the pre-defined role in template
    • For the above case, the person responsible need to be selected before user is allowed to submit the process. Else, "Submit" button will be disabled and tab will be shown in amber color for attention
    • You could also customize the frequency of reminder by clcking on the "Bell" icon at the tab
      • Reminder wll be sent if there is no action carried out by the responsible person after period of time

Submit Process

  1. Click on "Save Draft" to save process as draft for later submission
  2. Click on "Submit" to submit process for action and approval
    • Email notification will be triggered to approver/action taker / acknowledger upon process submission
    • Submit button will be enabled once all the mandatory info is provided

Act on Process (via Portal)

  1. From the received email, the approver/action taker can click on link in email to open the task in BPM
    • Alternatively, user can access thier pending task via "My Task" function in BPM
  1. Upload attachement (if any)
  2. Enter comment (mandatory for Rework or Reject action)
    • There might be other field(s) for input - subject to template configuration
  3. Click to perform action
    • For approval, user can perform :
      • Approve - Upon approve, process will continue to next step
      • Reject - Upon reject, initiator get notified and process end as rejected
      • Rework - Upon rework, selected routing person get notified and process will be routed back to him/her for action
    • For action taker, user can perform :
      • complete - Upon complete, process will continue to next step
      • Rework - Upon rework, selected routing person get notified and process will be routed back to him/her for action

Act on Process (via Email)

  1. From the received email, the approver/action taker can click on available action link in email
    • Email action might not be available for certain process. This is subject to configuration in template
  2. Upload attachement (if any)
  3. indicate comment (mandatory for reject and rework action)
  4. Click on "Reply" button
    • Once processed by BPM , email will be triggered to user to inform email action processing status (i.e. success or fail)
