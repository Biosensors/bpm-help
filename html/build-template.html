Overview

The build template function allow you to design the form with different type of input fields, configure who need to be involved in the process (e.g. action taker, approver or acknowledher) and how process should be routed for action.

Create a new template

  1. Click on "Build Template" menu link
  2. Click on "Create a new template" button
  1. Enter Template Name
  2. Enter Template Description
    • This is useful for requestor to know what is this process template for when they raise the process
  3. Enter Prefix
    • This will be prefix in process ID for process submitted under this template.
    • Must be unique and not used by other process tempalte
  4. Click on "Initiator" Section to start design your form
  5. Click on "Add Field" button
    • Upon click, field will be added under "Initiator" Section" for configuration
  6. On the newly added field, click on "Configuration" icon to start configure input field - see next section "Configure Field"
    • You could remove the field by click on "Delete" icon
    • You could also change the field's position with other fields through drag and drop

Configure field

  1. Please find below section on how to configure field for following input type :
    • text
    • select
    • radio
    • number
    • email
    • url
    • user-info
    • textarea
    • multi-select
    • checkbox
    • date
    • static-html
    • editable-html

Input Type : Text / Textarea / URL / Email / Number

  1. Select Input Type
  2. Check if field is mandatory
  3. Check if field is conditional input
    • Field is displayed only when a particular criteria is met . Else , it will be hidden from display
    • Refer "Conditional Rules" section for more details
  4. Enter "Label" of the field
  5. Enter "Pattern" of the field (optional)
    • Not applicable to "Number" input type
    • The data entered by initiator must meet this pattern. Else, error message will be shown
    • Please seek IT for assistance and configuration if pattern is required.
  6. Enter min. length of the field (optional)
  7. Enter max. length of the field (optional)

Input Type : Select / Radio / Checkbox / Multi-Select

  1. Select Input Type
  2. Check if field is mandatory
  3. Check if field is conditional input
    • Field is displayed only when a particular criteria is met . Else , it will be hidden from display
    • Refer "Conditional Rules" section for more details
  4. Enter "Label" of the field
  5. Configure selection list value
    • Static (default) - Configure by enter static list value
    • SQL Query - Configure by retrieve list value from in-house system database (Contact IT For assistance if you want to use this method)
    • Web Service Call - Configure by retrieve list value from external source via web service (Contact IT For assistance if you want to use this method)

Input Type : User Info

  1. Select Input Type
  2. Check if field is conditional input
    • Field is displayed only when a particular criteria is met . Else , it will be hidden from display
    • Refer "Conditional Rules" section for more details
  3. Enter "Label" of the field
  4. Select "User Info" to display
    • You may select to display initiator's profile info such as first name, last name, job title , cost center and etc

Input Type : editable-html / static-html

  1. Select Input Type
  2. Check if field is conditional input
    • Conditional field mean field is displayed only when a particular criteria is met . Else , it will be hidden from display
    • Refer "Conditional Rules" section for more details
  3. Enter "Label" of the field
  4. Enter content in html editor
    • Static-html - content will be displayed as read only to initiator
    • Editable-html - content will be displayed in editable html to initiator. When you have complicated form that cannot be built in BPM, you may consider to define the form in html editor and let initiator to fill in html format.

Configure Field Conditional Rules

All input type supports conditional rules. To enable conditional rules, check on the "Is Input Conditional" checkbox and follow the steps below for configuration.
  1. Choose Matching Field
    • You may use any of the "Select" or "Radio" field that you define in this template as matching field
  2. Choose Operator
  3. Choose Condition Value
    • Field will be shown to user when the define condition met. Else . field will be hidden to user

Configure Approver/Action Taker and Acknowledger

You are allowed to configure multiple approver, action taker and acknowledger in any sequence as per your business needs. Below are the steps on how to add and configure Approver/Action Taker and Acknowledger
  1. Click on "Add" button on Approver / Acknowledger / Action Taker panel
    • Approver allows to approve,reject and send back process for rework
    • Action taker allows to complete process and send back process for rework
    • No action required from Acknowledger. Acknowledger will get notified through email when proces routed to them.
  2. New section wiill be added automatically into configuration page
    • You could delete the section by clicking on the "Delete" icon
    • You could arrange the process sequence simply by drag and drop the section
    • You could add additional input field(s) by clicking on "Add" Field icon in panel
  3. Click on the "Config" icon to define role who needed to take action
  1. Select "Role Type"
    • Anyone in Biosensors
      • This allow initiator to select approver/action taker/acknowledger themselves the time when they initiate process
    • Spesific Role
      • Process will be routed to spesific role confgured at Company,Functions,Department and Job Title level for action
    • Functional Role (e.g. Direct Manager of / Function Head of) of Initiator / Approver / Action Taker / Acknowledger
      • Process will be routed for action as per configuration
    • Group Email
      • This option is applicable to Acknowledger only
      • It provide flexibility to inform group users via group email without needed to setup multiple acknowledger that target different users
  2. Check if role is conditional
    • Conditional role mean role is routed for action only when a particular criteria is met . Else , it will be not be routed for action
    • The configuration is similiar as Field Conditional Rules. Refer "Configure Field Conditional Rules" for details

Configure Reminder

Reminder setting allow you to configure how frequent the notification should be sent if there is no action from Approver / Action Taker. To configure reminder, select a section and click on the "Bell" icon.
  1. Enable/Disable Email Reply Allowed
    • When enable, system allow user to act on request by replying through email. If disabled, user has to act on the request through BPM system
  2. Enter reminder interval days
    • Reminder will be sent based on the configured interval days.
  3. Enter Number of Times
    • Reminder will stop send to user once reach the configured value

Submit Template for Approval

  1. Click on "Preview" button to review template before submission
    • You will be able see how the form is rendered to requestor and who will be routed to for action and approval
  2. Click on "Save as Draft" for later submission
    • This is pretty useful when process is complicated and require time to design
  3. Click on "Submit" button to submit the template for approval
    • Template will be first routed to your local IT, follow by your functional head for review and approval
    • Template will be released and avaialble to all users to raise process once approval given by both local IT and functional head
    • Template will be locked for amendment once it get approved. For template changes, you could clone the existing template, make changes and submit for approval again. You should disabled your template once the new template is released.
