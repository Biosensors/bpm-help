Overview

The build template function allow you to design the form with different type of input fields, configure who need to be involved in the process (e.g. action taker, approver or acknowledger) and how process will be routed for action.

Create a new template

  1. Click on "Build Template" menu link
  2. Click on "Create a new template" button
  3. Note : The "New template from copy" button located at the bottom-right side is used to copy template from Test Enviroment to Live Enviroment. Do contact IT for assistance for more details about the usage.
  1. Enter Template Name
    • Please refer WI-11719 for the standard naming convention
  2. Enter Template Description
    • This is useful for requestor to know what is this process template for when they raise the process
  3. Enter Prefix
    • This will be prefix in process ID for process submitted under this template.
    • Must be unique and not used by other process template
  4. Indicate if on-behalf user confirmation is required
    • If yes, system will route to on-behalf person for confirmation if process is raised on behalf of them by someone else.
  5. Indicate if scheduled start date allowed
    • If yes, system will allow initiator to indicate process start date during process submission.
    • The submitted process will start route for action only on the scheduled start date.
  6. Indicate if duplicate approver allowed
    • If no, system bypass and routed for approval only once. (e.g. if xxx template is configured to route to direct manager and HOD for approval. if direct manager and HOD is the same person, system bypass direct manager and only route to HOD for approval.
  7. Click on "Initiator" Section to start design your form
    • Note : Form can be designed in other steps as well (such as approver, action taker and etc)
  8. Click on "Add Field" button to add field control
    • Upon click, field control will be added under "Initiator" Section" for configuration
    • Refer "Configure field" section for more details on the configuration
  9. Click on "Add Table" button to add table control
    • Upon click, table control will be added under "Initiator" Section" for configuration
    • Refer "Configure table" section for more details on the configuration

Configure field

  1. On the newly added field, click on "Configuration" icon to start configure input field - see next section "Configure Field"
    • You could remove the field by click on "X" icon
    • You could change the field's position with other fields through drag and drop
    • You could clone the field as new field for usage through the copy icon next to the "Config" icon
  2. Please find below section for the configuration for different control type


Input Control Type : Text / Textarea / URL / Email / Number

  1. Select "Control Type" as Input
  2. Select "Input Type" as Text / Textarea / URL / Email / Number
  3. Check if field is mandatory
  4. Enter "Label" of the field
  5. Enter "Pattern" of the field (optional)
    • Not applicable to "Number" input type
    • The data entered by initiator must meet this pattern. Else, error message will be shown
    • Please seek IT for assistance and configuration if pattern is required.
  6. Enter min. length of the field (optional)
  7. Enter max. length of the field (optional)
  8. Select data source for initial value
    • This is a advance function used to pre-populate value using data from other system.
    • Please seek IT assistance for configuration if required
  9. Check if field is conditional input
    • Field is displayed only when a particular criteria is met . Else , it will be hidden from display
    • Refer "Conditional Rules" section for more details

Configure Field Conditional Rules

  1. To enable conditional rules, check on the "Is Field Conditional" checkbox and follow the steps below for configuration.
    • All control type supports conditional rules.
  2. Choose Matching Field
    • You may use any field in form that you define in this template as matching field
  3. Choose Operator (e.g.= equal, no value, value exits and etc)
  4. Choose Condition Value
    • Field will be shown to user when the define condition and value matched. Else . field will be hidden to user
  5. Click "+" icon to add more condition (if needed)

Configure Workflow

You are allowed to configure workflow with multiple approver, action taker, acknowledger, user group and automator in any sequence as per your business needs.

  • Approver : Allow to approve,reject and send back process for rework
  • Action taker : Allow to complete process and send back process for rework
  • Acknowledger : Be notified only. Not action required.
  • User group : Mutliple approver/action taker can be added in the user group with setting either all require to take action or anyone from the group to take action
    • User under the same group will receive task simultaneously
  • Automator : Pre-defined System automated process without human intervention (e.g. integration with other system for data exchange)
    • Please seek IT assistance for configuration if required


Below are the steps on how to add and configure steps into workflow.
  1. Click on "Add" button on Approver / Acknowledger / Action Taker / Automator / User Group panel
  2. New section wiill be added automatically into configuration page
    • You could delete the section by clicking on the "Delete" icon
    • You could arrange the process sequence simply by drag and drop the section
    • You could add additional input field(s) by clicking on "Add" Field icon in panel
    • You could also clone the section via the "copy" icon
  3. For each added section, click on the "Config" icon to define role who needed to take action


Appover Configuration

  1. Select "Role Type"
    • Anyone in Biosensors
      • This allow initiator to select own approver from user listing during process initiation
    • Spesific Role
      • Process will be routed to spesific role confgured at Company,Functions,Department and Job Title level for action
      • The role can be setup in 4 ways
        • Company
        • Company + Function
        • Company + Function + Department
        • Company + Function + Department + Job Title
      • If there is more than one person matched for the define role , system will display as user list for user selection
    • Same User As
      • Process will be routed to person same as in other user steps (user to specify)
    • Requestor
      • Process will be routed to requestor
    • Direct Manager Of
      • Process will be routed to direct manager of the user in other user steps (user to specify)
    • Function Manager Of
      • Process will be routed to Function Manager of the user in other user steps (user to specify)
    • Function Head Of
      • Process will be routed to Function Head of the user in other user steps (user to specify)
    • Function VP Of
      • Process will be routed to Function Head of the user in other user steps (user to specify)
    • CFO
      • Process will be routed to Company CFO
    • CEO
      • Process will be routed to Company CEO
  2. Choose Validator
    • This allow system to perform additional validation upon action done by user (e.g. check if mandatory data is provided and etc)
    • Please seek IT assistance for configuration if required
  3. Check if Input conditional
    • Conditional role mean role is routed for action only when a particular criteria is met . Else , it will be not be routed for action
    • The configuration is similiar as Field Conditional Rules. Refer "Configure Field Conditional Rules" for details

Configure Reminder

Reminder setting allow you to configure how frequent the notification should be sent if there is no action from Approver / Action Taker. To configure reminder, select a section and click on the "Bell" icon.
  1. Enable/Disable Email Reply Allowed
    • When enable, system allow user to act on request by replying through email. If disabled, user has to act on the request through BPM system
  2. Enter reminder interval days
    • Reminder will be sent based on the configured interval days.
  3. Enter Number of Times
    • Reminder will stop send to user once reach the configured value

Submit Template for Approval

  1. Click on "Preview" button to review template before submission
    • You will be able see how the form is rendered to requestor and who will be routed to for action and approval
  2. Click on "Save as Draft" for later submission
    • This is pretty useful when process is complicated and take time to design
  3. Click on "Submit" button to submit the template for approval
    • Button will be disabled if system detect there is missing mandatory information or invalid setting
    • Upon submission, template will first be routed to your local IT, follow by your functional head for review and approval
    • Template will be released and avaialble to all users to raise process once approval given by both local IT and functional head
  4. Click on "Copy Template" button to copy template from dev enviroment to production enviroment
    • Please contact IT for assistance for this exercise
  5. Click on "Reset" button will erase current template configuration and start as new template
